---

## このプロジェクトについて

### 何をするプロジェクトか

既存の Vue2 + Vuetify2 + Vue CLI で構築されたSPAを、**Vue3 + TypeScript + Composition API** へ移行するプロジェクトです。

### 移行にあたって導入する新しいもの

| 技術 | 役割 | Vue2時代との違い |
|------|------|------------------|
| Composition API | コンポーネントの書き方 | Options API から変更 |
| TypeScript | 型安全な開発 | JavaScript から変更 |
| Vite | ビルドツール | Vue CLI から変更 |
| Pinia | 状態管理 | Vuex から変更 |
| Storybook | UIカタログ・確認ツール | 新規導入 |
| MSW | APIモック | 新規導入 |

## 心がけてほしいこと

### Vue2時代との違いを受け入れる

Vue2プロジェクトでは許容されていた書き方が、Vue3 + CDDでは禁止されていることがあります。

| Vue2時代 | Vue3 + CDD |
|----------|------------|
| コンポーネント内でAPIを直接呼ぶ | ❌ 禁止 |
| `this.$store` で状態を直接参照 | ❌ 禁止 |
| Options API の `data`, `methods` | ❌ 使わない |
| 画面を作ってから部品に分ける | ❌ 逆の順番 |

最初は違和感があるかもしれませんが、この制約がチーム開発をスムーズにします。

### 「小さく作って組み合わせる」

このプロジェクトの基本思想です。

- 画面全体を一気に作らない
- 部品（コンポーネント）を先に作る
- 部品を Storybook で確認してから画面に組み込む
