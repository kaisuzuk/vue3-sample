# 用語集

このドキュメントでは、Vue3 マイグレーションプロジェクトで使用する用語を説明します。
ドキュメントや会話で意味がわからない言葉があったら、ここを参照してください。

---

## プロジェクト・チーム関連

| 用語 | 説明 |
|------|------|
| **BP** | ビジネスパートナー。プロジェクトに参加する外部メンバーのこと。 |
| **マイグレーション** | 移行のこと。このプロジェクトでは Vue2 から Vue3 への移行を指す。 |

---

## Vue 関連

| 用語 | 説明 | Vue2 での対応 |
|------|------|--------------|
| **Composition API** | Vue3 で推奨されるコンポーネントの書き方。`setup()` 関数や `<script setup>` を使う。 | Options API |
| **Options API** | Vue2 で使っていた書き方。`data`, `methods`, `computed` などのオプションを使う。 | - |
| **`<script setup>`** | Composition API の簡略記法。`export default` が不要になる。 | なし |
| **ref** | リアクティブな値を作る関数。`.value` でアクセスする。 | `data()` |
| **reactive** | リアクティブなオブジェクトを作る関数。 | `data()` |
| **computed** | 他の値から導出される値を定義する関数。 | `computed: {}` |
| **watch** | 値の変化を監視する関数。 | `watch: {}` |
| **defineProps** | コンポーネントが受け取る props を定義するマクロ。 | `props: {}` |
| **defineEmits** | コンポーネントが発火するイベントを定義するマクロ。 | `emits: []` |
| **Pinia** | Vue3 推奨の状態管理ライブラリ。 | Vuex |
| **Vite** | Vue3 推奨のビルドツール。高速な開発サーバーが特徴。 | Vue CLI |

---

## CDD（設計）関連

| 用語 | 説明 |
|------|------|
| **CDD** | Component Driven Development（コンポーネント駆動開発）。画面ではなくコンポーネントを最小単位として開発する考え方。 |
| **pages** | ルーティングの入口となるコンポーネント。画面遷移を制御する。 |
| **widgets** | 画面の骨組みを担当するコンポーネント。状態と sections をつなぐ役割。 |
| **sections** | 意味のある入力・表示の塊。props と emit だけで動作する。 |
| **shared/ui** | アプリ全体で使い回す共通 UI 部品（Button, TextField など）。 |
| **model** | 状態とビジネスロジックを管理する層。`useXxx.ts` として実装する。 |
| **usecase** | model と同義。画面固有の状態・ロジックを管理する Composable。 |
| **entities** | ドメインロジックを担当する層。純関数のみ。 |
| **Wrapper** | shared/ui を画面固有の用途に合わせてラップしたコンポーネント。 |

---

## データフロー関連

| 用語 | 説明 |
|------|------|
| **props** | 親コンポーネントから子コンポーネントにデータを渡す仕組み。 |
| **emit** | 子コンポーネントから親コンポーネントにイベントを通知する仕組み。 |
| **props down, events up** | データは props で下に流し、変更は emit で上に戻すというルール。 |
| **actions** | 状態を更新するための操作（メソッド）。model が公開する。 |
| **view** | 表示用に加工された導出値。model が computed で公開する。 |
| **form** | フォーム入力値。model が管理する「入力の真実」。 |

---

## API・型定義関連

| 用語 | 説明 |
|------|------|
| **DTO** | Data Transfer Object。API が返すデータ構造をそのまま表現した型。 |
| **ViewModel** | UI 表示用に加工されたデータの型。DTO から変換して作る。 |
| **Swagger** | API 仕様書を自動生成するツール。OpenAPI 仕様に基づく。 |
| **OpenAPI** | REST API を記述するための標準仕様。 |
| **contracts** | API の型定義（DTO）を配置するディレクトリ。 |
| **services** | API 呼び出しを担当するサービス層。 |

---

## MSW 関連

| 用語 | 説明 |
|------|------|
| **MSW** | Mock Service Worker。API 通信をモック（偽装）するライブラリ。 |
| **handler** | 1 つの API エンドポイントに対する振る舞いの定義。 |
| **fixture** | モックで返すデータ。API レスポンスそのものの形式で定義する。 |
| **scenario** | handler のセット。正常系、エラー系などの状況別に定義する。 |
| **Service Worker** | ブラウザとネットワークの間で動作するスクリプト。MSW はこれを使って通信を横取りする。 |

---

## Storybook 関連

| 用語 | 説明 |
|------|------|
| **Storybook** | コンポーネントを単体で確認・開発できるツール。 |
| **Story** | コンポーネントの 1 つの状態を表す定義。 |
| **args** | Story に渡す props の値。 |
| **autodocs** | Storybook が自動生成するドキュメント。 |

---

## 状態・バリデーション関連

| 用語 | 説明 |
|------|------|
| **loading** | データ読み込み中の状態。 |
| **submitting** | フォーム送信中の状態。 |
| **touched** | ユーザーが一度でも操作したかどうかのフラグ。 |
| **dirty** | 初期値から変更があったかどうかのフラグ。 |
| **errors** | バリデーションエラーの内容。 |
| **canSubmit** | 送信可能かどうかを示すフラグ。 |

## その他

| 用語 | 説明 |
|------|------|
| **Composable** | Composition API で作る再利用可能なロジック。`useXxx` という命名規則。 |
| **純関数** | 同じ入力に対して常に同じ出力を返し、副作用を持たない関数。 |
| **副作用** | 関数の外部に影響を与える処理（API 呼び出し、DOM 操作、グローバル変数の変更など）。 |
| **リアクティブ** | 値が変更されたときに自動的に UI が更新される仕組み。 |
| **SPA** | Single Page Application。ページ遷移なしで画面を切り替えるアプリケーション。 |
| **E2E** | End to End。実際のユーザー操作をシミュレートするテスト。 |
| **Playwright** | E2E テストを行うためのツール。 |

## Vue2 → Vue3 用語対応表

| Vue2 | Vue3 |
|------|------|
| `data()` | `ref()` / `reactive()` |
| `this.xxx` | `xxx.value` |
| `computed: {}` | `computed(() => ...)` |
| `methods: {}` | 通常の関数 |
| `watch: {}` | `watch()` / `watchEffect()` |
| `props: {}` | `defineProps()` |
| `emits: []` | `defineEmits()` |
| `created()` | `<script setup>` 内に直接書く |
| `mounted()` | `onMounted()` |
| `beforeDestroy()` | `onBeforeUnmount()` |
| `destroyed()` | `onUnmounted()` |
| Vuex | Pinia |
| `this.$store` | `useXxxStore()` |
| `this.$emit()` | `emit()` |
| `this.$router` | `useRouter()` |
| `this.$route` | `useRoute()` |
